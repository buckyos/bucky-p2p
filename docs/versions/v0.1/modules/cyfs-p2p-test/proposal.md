# cyfs-p2p-test 提案

## 背景与目标
- `cyfs-p2p-test` 是工作区的运行时场景驱动器。
- 目标是让场景、CLI 和配置流改动纳入同样严格的分阶段流程，以保证运行时验证仍然可信。

## 范围
### 范围内
- CLI 子命令和运行时场景行为
- 配置加载和本地制品使用
- 作为 DV 证据面的运行时验证职责

### 范围外
- 拥有适配层或核心协议语义
- 用临时本地运行替代正式模块测试计划

### 与相邻模块的边界
- `cyfs-p2p-test` 负责证明运行时行为，但不重新定义 `cyfs-p2p` 或 `p2p-frame`。

## 约束
- 保持场景入口明确
- 记录本地制品前置条件
- 不允许场景便利性掩盖模块级测试缺口

## 高层结果
- 运行时 harness 改动变得可审计
- 场景语义持续与依赖它的模块数据包保持一致

## 验收锚点
| 条目 ID | 结果描述 | 约束/非目标 | 验收如何判断 |
|---------|----------|-------------|--------------|
| P-1 | 运行时 harness 改动可审计 | 不拥有核心协议设计 | proposal/design/testing/acceptance 证据链齐全 |
| P-2 | 场景语义与依赖它的模块数据包保持一致 | 不用临时运行替代正式测试计划 | testing 和 testplan 保留 DV / integration 证据 |

## 风险
- 对环境敏感而导致的不稳定
- 配置漂移
- 由未治理的临时运行造成的虚假信心
