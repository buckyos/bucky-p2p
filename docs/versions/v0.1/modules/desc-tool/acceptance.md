# desc-tool 验收标准

## 验收基线
- 成功与否以已批准的 `proposal.md` 为准。

## 必需证据
- 已批准的 `proposal.md`
- 已批准的 `design.md`
- 已批准的 `testing.md`
- `testplan.yaml`
- `docs/modules/desc-tool.md`
- 影响输出或签名行为的变更证据

## 验收标准
- 子命令职责保持明确
- 输出和签名行为拥有已声明的验证证据
- 评审结论中明确点出了安全敏感范围

## 失败条件
- CLI 或输出行为发生变化，但没有面向操作员的证据
- 签名或文件格式变更缺少更强的评审关注
- 输出语义偏离已批准范围
